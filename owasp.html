Owasp Top 10 Security Risk
Posted By Oluwaferanmi Olatunji
OWASP Top 10 - The Open Web Application Security Project, is an NGO aimed at improving the security of software applications. OWASP top 10 is a standard awareness document for developers and web application security.
    Owasp Top 10 Security Risk
  • Broken Access Control - May occur when access is breached by unauthorized users
    Prevention
    Except for public resources, deny by default.
    Implement access control mechanisms once and re-use them throughout the application.
    Log access control failures, alert admins when appropriate
  • Cryptographic Failures - Occurs when sensitive date e.g credit cards, passwords are not properly encrypted.
    Prevention
    Don't store sensitive data unnecessarily. Make sure to encrypt all sensitive data at rest.
    Disable caching for responses that contain sensitive data.
    Apply required security controls as per the data classification.
  • Injection - A vulnerability that allows attackers to get data in a malicious way from an app
    Prevention
    Use positive server-side input validation.
    Use a safe API, which avoids using the interpreter entirely.
  • Insecure Design - When Secure Development Life Cycle is not thoroughly followed in the development process, it may threaten the app’s security.
    Prevention
    Create and use a secure development lifecycle that obeys design security and privacy-related controls
    Integrate security language and controls into user stories
  • Security Misconfiguration - The software may be vulnerable if unnecessary features are installed, software is out of date, misconfigured permission on cloud services
    Prevention
    Regular review to check and update configurations
    Remove unused dependencies / unnecessary features
  • Vulnerable And Outdated Components - A software may be vulnerable if software developers do not test it regularly, check the compatibility of updated or patched libraries
    Prevention
    Remove unused dependencies / unnecessary features
    Only obtain components from official sources over secure links
  • Identification And Authentication Failures - Occurs when there is an authentication weakness.
    Prevention
    Implement weak password checks, such as testing new or changed passwords against the top 10,000 worst passwords list.
    Use a server-side, secure, built-in session manager that generates a new random session ID with high entropy after login.
    Where possible, implement multi-factor authentication to prevent automated credential stuffing, brute force, and stolen credential reuse attacks.
  • Software And Data Integrity Failures - This occurs when there is a breach in the integrity of software or data
    Prevention
    Use hashing / digital to protect software’s integrity
    Ensure code is reviewed before deployment
  • Security Logging And Monitoring Failures - Without logging and monitoring, breaches cannot be detected. Insufficient logging, detection, monitoring, and active response occurs any time:
    Logs of applications and APIs are not monitored regularly
    Logs are only stored locally
    Prevention - Developes should ensure ;
    Ensure that logs are generated in a format that log management solutions can easily consume.
    Ensure log data is encoded correctly to prevent injections or attacks on the logging or monitoring systems.
  • Server Side Request Forgery - this occurs whenever a web application is fetching a remote resource without validating the user-supplied URL.
    Prevention
    Ensure thorough validation before authentication
    Do not send raw responses to clients
    Disable HTTP redirections
    Don't deploy other security relevant services on front systems
